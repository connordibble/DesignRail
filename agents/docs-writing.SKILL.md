# Docs Writing Skill

## Use when

Work changes README content, architecture notes, generated reports, or portfolio narrative.

## Goal

Make the project understandable to hiring managers and senior engineers.

## Rules

- Explain that the review UI is the product.
- Explain tradeoffs.
- Keep claims honest.
- Avoid AI hype.
- Include architecture diagrams where useful.
- Make setup commands accurate.
- Keep docs concise but substantive.
- Prefer public-safe examples over company-specific references.
- Explain where deterministic checks enforce quality.
- Document mock mode before optional real service paths.
