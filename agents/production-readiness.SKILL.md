# Production Readiness Skill

## Use when

Work assesses what is missing for CI, security, governance, or real design-tool integration.

## Goal

Evaluate what would be needed to move DesignRail from local/internal tooling to production-grade internal platform.

## Cover

- Real Figma API/MCP integration
- Auth and permissions
- CI/CD integration
- Merge request comments
- GraphQL schema governance
- Database migrations and backups
- Observability
- Adoption metrics
- Human-in-the-loop review
- Design-token governance
- Security review

## Rules

- Separate demo behavior from production requirements.
- Do not add real service dependencies to the default mock path.
- Identify risks, owners, and verification gates.
