# Component Mapping Skill

## Use when

Work selects Shoelace components or maps normalized intent to reviewable implementation recommendations.

## Goal

Map normalized design intent to Shoelace Web Component usage for human review.

## Rules

- Prefer existing Shoelace components over custom implementations.
- Do not invent unsupported variants.
- Use explicit fallback mappings.
- Produce confidence, rationale, and review notes for every mapping.
- Document any mapping ambiguity.
- Keep mappings component-library agnostic where practical.
- Map design tokens to CSS custom properties explicitly.
- Include accessibility requirements in the mapping output.
- Do not mark a mapping export-ready until a human accepts or edits it.
