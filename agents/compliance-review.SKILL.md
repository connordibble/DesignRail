# Compliance Review Skill

## Use when

Work evaluates design-system alignment, accessibility, token usage, or release readiness.

## Goal

Produce deterministic, senior-level design-system compliance feedback.

## Required sections

- Summary
- Accessibility
- Token usage
- Variant coverage
- Human review status
- React integration readiness
- GraphQL contract readiness
- Documentation readiness
- Design-system alignment
- Blocking issues
- Suggested fixes
- Approval status

## Severity

- Blocker
- Warning
- Info

## Rules

- Explain tradeoffs and remediation steps.
- Do not imply automated approval replaces human review.
- Make approval status explicit: approved, approved with warnings, or blocked.
- Flag missing confidence, rationale, accessibility metadata, or review decision history.
