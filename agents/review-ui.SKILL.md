# Review UI Skill

## Use when

Work changes React screens, review flows, decision controls, comparison views, or exports.

## Goal

Build a practical internal UI where developers review, edit, accept, reject, and export mappings.

## Rules

- Show source intent, recommended mapping, compliance findings, confidence, and rationale together.
- Support accept, reject, edit, and export-ready states.
- Persist decisions through GraphQL.
- Make loading, empty, error, disabled, pending, and saved states visible.
- Keep layouts scannable for repeated review work.
- Use accessible controls, keyboard-friendly interactions, and clear focus states.
